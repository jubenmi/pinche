# 相册整册朋友圈使用首张可用媒体封面设计

状态：用户已确认

日期：2026-07-24

## 1. 目标

以最小改动替换相册整册朋友圈的 Canvas 拼图：从当前分享范围中固定选择第一张可用
媒体作为封面。本方案不引入 Skyline，不迁移相册页，不增加服务端截图能力。

好友和群聊整册分享继续省略 `imageUrl`，使用微信自动截取的当前相册页面。朋友圈
整册分享继续提供明确 `imageUrl`，但该图片直接来自相册媒体，不再经过 Canvas 合成。

## 2. 明确不做

- 不把 `pages/session/album` 或任何新增页面改成 Skyline。
- 不使用 `<snapshot>`。
- 不增加无头浏览器、服务端 HTML 截图或新的图片生成服务。
- 不生成拼图、海报、票根或静态兜底图。
- 不修改整册分享 token、隐私、审核、撤销和过期协议。
- 不改变单媒体、招募、首页、游后感和 fail-closed 分享策略。

## 3. 封面选择规则

整册朋友圈使用确定性的“第一张可用媒体”，不在每次分享时随机选择，以避免同一 token
的封面反复变化。

候选顺序使用后端 `cover_recipe.images` 的既有顺序。该配方已经按当前分享 token 的
公开范围筛选，并且最多包含三张候选图。选择过程按顺序检查候选：

1. 如果候选媒体在当前页面已有可用本地图片路径，直接使用本地路径。
2. 否则使用候选的公开安全 `thumbnail_url`。
3. 当前候选没有本地路径且没有线上缩略图时，继续检查下一项。
4. 第一张得到可用路径的候选立即成为朋友圈封面。
5. 所有候选均不可用时，整册朋友圈保持不可分享；不得回退到 Logo、Canvas 或静态图。

“本地路径”仅接受微信临时文件或小程序本地临时目录，例如 `wxfile://`、
`http://tmp/`、`file://`、`/tmp/`、`/private/`、`/var/`。静态资源路径和远程 URL
不能伪装成本地优先项。

线上路径必须来自当前 token 的公开安全 `thumbnail_url`，并通过现有媒体 URL
规范化逻辑处理。不得从成员私有照片列表拼接未授权 URL。

视频候选继续使用后端提供的安全封面/缩略图，不加载视频文件本身。

## 4. 三条整册链路

### 4.1 成员默认整册

默认分享 token 准备完成后，从其 `cover_recipe` 选择第一张可用媒体。本地优先映射
只允许使用 recipe 中同一媒体 ID 对应的当前页面预览。朋友圈 payload 使用默认 token
及该 token 的代表图。

### 4.2 成员主动“全部/选中”

创建主动 token 后，从该响应自己的 `cover_recipe` 选择代表图。选中范围打开时必须
使用主动 token 和主动代表图，不得借用默认整册封面。关闭主动分享层后恢复默认整册
状态。

### 4.3 公开整册再次分享

公开相册加载后，从公开响应的 `cover_recipe` 选择代表图。公开页通常没有成员本地
预览，因此直接使用第一张可用的线上安全缩略图。

## 5. 分享菜单与 payload

- 整册好友/群聊：有效 token 就绪后开放，payload 省略 `imageUrl`。
- 整册朋友圈：有效 token 和代表图均就绪后开放，payload 的 `imageUrl` 为所选本地
  路径或线上安全缩略图。
- 代表图准备失败：只隐藏朋友圈，好友/群聊仍可分享。
- `onShareTimeline` 被异常调用但没有当前代表图时返回不可分享状态，不提供静态兜底。
- 成员主动分享层打开时，朋友圈优先使用主动状态；关闭后恢复默认状态。

单张照片和单个视频继续使用对应媒体的明确封面，不经过整册代表图选择器。

## 6. 实现边界

新增一个框架无关的小工具，负责：

- 按 recipe 顺序选择第一张可用媒体；
- 按媒体 ID 查询本地路径；
- 本地不可用时规范化线上 `thumbnail_url`；
- 返回空字符串表示没有可分享封面。

相册页复用现有 token 准备流程和请求代次控制，只把 Canvas 准备步骤替换为代表图
选择。保留现有迟到请求拒绝逻辑，避免旧 token 的封面覆盖新 token。

兼容验证通过后删除：

- 相册页两个隐藏 Canvas；
- `albumShareCanvas.js` 及其专用测试；
- Canvas runtime、节点查询、绘制、导出、临时文件回收和协调器；
- 整册 Canvas context key、prepared 任务与静态 fallback；
- 只验证整册 Canvas 拼图的静态门禁。

后端 `cover_recipe` 保留，因为本方案直接使用其候选顺序和公开缩略图。

## 7. 编译器与页面配置回退

此前为 Skyline 试验升级的三项 uni-app 编译器依赖恢复到项目原版本：

```text
3.0.0-5000720260410001
```

从 `pages/session/album` 移除 `renderer: skyline` 和
`componentFramework: glass-easel`。其他页面也不得新增这些设置。

## 8. 测试

自动化至少验证：

1. 第一候选有本地路径时选择本地路径。
2. 本地缺失时选择同一候选的线上缩略图。
3. 第一候选完全不可用时选择下一候选。
4. 不接受远程 URL 或静态资源路径冒充本地路径。
5. 全部不可用时返回空字符串。
6. 默认、主动和公开整册各自使用自己的 token 与代表图。
7. 整册好友 payload 继续省略 `imageUrl`。
8. 没有代表图时朋友圈菜单隐藏且没有静态兜底。
9. 单媒体、招募和 fail-closed 明确封面保持不变。
10. 相册页不包含 Skyline、`<snapshot>`、Canvas 节点或 Canvas 导出代码。
11. D55/D56、相册分享 Node 测试、小程序结构检查和生产构建全部通过。

## 9. 验收标准

1. 相册页继续使用原 WebView 渲染，复杂交互不需要迁移。
2. 整册好友/群卡片继续展示微信截取的当前相册页。
3. 整册朋友圈封面展示当前分享范围中的一张真实相册媒体。
4. 本地图片可用时优先使用本地路径，否则使用公开安全线上缩略图。
5. 不再生成整册 Canvas 拼图，也不引入 Skyline。
6. 无可用媒体时不开放朋友圈，且不显示 Logo 或静态兜底。
7. 单媒体和其他分享类型行为保持不变。

# 微信开发者工具最新产物与改期回归防护设计

## 背景与根因

微信开发者工具项目固定读取 `apps/miniprogram/dist/dev/mp-weixin/`。当前该目录的入口和车局管理页面产物生成于 2026-07-16 22:48，而改期功能合并、北京时间展示修复和 API 数据库 UTC 修复分别在后续提交中落地。源码已经包含统一 `Asia/Shanghai` 工具和改期入口，但开发者工具仍打开旧产物。

旧产物把 `2026-07-18T05:00:00Z` 的 `05:00` 前缀当作本地墙上时间：日历因此显示 `05:00`；管理页也把车局误判为已开始，隐藏“改期”。现有 `devtools-refresh-hook.js` 只检查输出文件是否存在，不检查它们是否覆盖当前源码，因此会重复打开过期包。

## 目标

1. 从当前 `develop` 源码重新生成包含全部现有功能的 `dist/dev/mp-weixin`。
2. 开发者工具刷新入口在产物旧于源码时关闭式拒绝打开，明确提示先构建。
3. 显式的“构建并刷新”命令负责生成最新开发包、确认核心产物完整，再打开微信开发者工具。
4. 在开发者工具内验证同一绝对时间按北京时间显示，且未发车车局显示并可打开改期选择器。
5. 保持生产 API、数据库数据、内容审核业务开关和改期业务规则不变。

## 方案比较

### 方案 A：开发包重建 + 过期门禁 + 开发者工具实测（采用）

保留 `dist/dev/mp-weixin` 作为开发者工具目录；补强现有刷新脚本的产物新鲜度判定，并提供一次显式重建入口。兼顾日常开发能力和防回归，改动范围集中在开发工具链。

### 方案 B：仅重新构建

可以立即恢复，但旧产物仍会被后续刷新脚本接受，同一问题可能再次发生。

### 方案 C：固定读取生产构建目录

生产构建可重复，但会削弱日常开发热更新和调试体验，不采用。

## 设计

### 1. 产物新鲜度边界

刷新脚本继续只观察小程序源码、共享时间模块、小程序配置和相关包配置。它必须同时满足以下条件才允许打开开发者工具：

- 既有必需文件完整且 JSON 可解析；
- 车局管理、日历、共享时间和入口产物均存在；
- 当前源码快照有对应的成功构建指纹；
- 指纹与当前被观察文件内容完全一致，而不只依赖文件存在或单个 mtime。

构建指纹只包含路径、内容哈希和生成时间，不包含环境变量、登录信息或业务数据。指纹文件位于 gitignored 的开发产物目录内。

### 2. 显式构建与刷新

`npm run devtools:refresh` 作为人工入口：

1. 启动一次小程序开发构建；
2. 等待 UniApp 报告首次构建完成并确认必需产物可读；
3. 写入当前源码指纹；
4. 停止本次一次性 watcher，避免残留重复进程；
5. 用微信开发者工具 CLI 打开 `dist/dev/mp-weixin`。

Codex Stop 自动刷新不自行构建；若指纹缺失或过期，只输出明确提示并拒绝打开旧包，避免无意启动长期 watcher。

### 3. 回归测试

先增加失败测试，再修改刷新脚本。测试至少证明：

- 仅有旧产物时，刷新判定为 stale；
- 完整产物但指纹不匹配时，刷新判定为 stale；
- 指纹匹配且必需文件完整时才判定为 ready；
- 北京时间工具把 `2026-07-18T05:00:00Z` 显示为 `2026-07-18 13:00`；
- 同一未来时间允许改期；
- 最终开发产物包含改期入口、专用接口和统一北京时间模块。

### 4. 开发者工具验收

清除编译缓存并重新编译后，使用现有登录态检查：

- 日历中的目标车局显示 `13:00`，不显示 `05:00`；
- 管理页显示北京时间；
- 发车前显示“改期”按钮；
- 点击“改期”能打开日期时间选择器和确认流程；
- 不在验收中提交真实改期，避免改变生产业务数据；后端专用接口继续由现有自动化测试覆盖。

若开发者工具登录态或生产车局状态不满足验证条件，使用不修改生产数据的本地/模拟测试车局，并记录该限制，不能用静态检查替代界面结论。

## 错误处理与安全

- 构建失败、超时、必需产物缺失或指纹不匹配时不打开开发者工具。
- 一次性 watcher 无论成功或失败都必须被终止。
- 不删除用户文件，不提交 `dist`，不改微信 Token，不改生产 Stack。
- 当前 `package-lock.json` 与 `docs/evidence/` 为用户已有未提交内容，实施过程必须保持不纳入提交。

## 完成标准

- 自动化回归测试经历 RED → GREEN，且现有北京时间与改期测试全部通过。
- 最新 `dist/dev/mp-weixin` 由当前源码生成，指纹匹配。
- 开发者工具实际显示北京时间和改期入口，控制台无新增运行时错误。
- 过期开发包以后不能再被刷新脚本静默打开。

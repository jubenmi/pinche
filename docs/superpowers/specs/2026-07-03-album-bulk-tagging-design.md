# 相册批量标注设计

日期：2026-07-03

## 结论

小程序车局相册增加多选批量标注能力。用户可以进入多选模式，勾选多张自己有权限标注的照片，然后选择一组人物标签；保存时前端逐张调用现有单张标注接口，把所有选中照片的标签替换为同一组标签。

本设计选择前端循环保存，不新增后端批量接口。这样保留现有权限校验、标签校验和“替换整组标签”的接口语义，改动范围最小。

## 背景

当前相册已经支持上传、瀑布流展示、按全部/我上传/有我/待标注过滤，以及对单张照片打开标注面板保存人物标签。标注接口是：

```text
PUT /api/session-album/photos/:photoId/tags
```

该接口接收 `tagKeys`，并替换该照片已有的整组标签。它已经负责校验照片是否存在、照片是否属于当前用户、车局相册是否开放、标签是否属于本车局。

新需求来自高频整理场景：例如 10 张照片都包含 A 角色时，用户不应该逐张打开同一个标注面板重复操作。

## 目标

1. 相册页提供多选入口。
2. 用户可以一次选择多张 `can_tag` 照片。
3. 用户可以对已选照片选择同一组标签。
4. 保存时把所有选中照片都替换为同一组标签。
5. 不新增后端接口，不改变现有相册隐私和权限模型。
6. 单张标注、删除、上传、筛选和预览行为保持兼容。

## 非目标

- 不做后端批量标注接口。
- 不做追加标签模式。
- 不做跨车局批量标注。
- 不允许标注非本人上传或后端返回 `can_tag = false` 的照片。
- 不改变照片可见性规则。

## 小程序交互

### 多选入口

相册页顶部操作区在可上传/可标注场景下增加“多选”按钮。进入多选模式后：

- 顶部按钮变为“退出多选”。
- 筛选仍然可用；切换筛选时退出多选模式并清空选择。
- 上传、隐私设置、删除和单张标注按钮在多选模式下不作为主操作展示，避免误触。

### 选择照片

照片卡片在多选模式下展示选择态：

- `can_tag = true` 的照片可以点选或取消。
- `can_tag = false` 的照片显示不可选状态，不加入选择集合。
- 选中照片显示明显勾选态。
- 未选中照片仍可看出图片和原有标签摘要。

点击图片区域在多选模式下优先切换选中状态，不打开图片预览。退出多选模式后恢复原预览行为。

### 批量标注动作

页面底部增加批量操作栏：

- 显示 `已选 N 张`。
- 提供“取消”和“批量标注”。
- 未选择照片时，“批量标注”不可用。

点击“批量标注”后复用现有人物选择面板：

- 标题改为 `给 N 张照片标注`。
- 人物分组继续使用车友、DM / NPC 工作人员、NPC 角色、其他。
- 默认不从某一张照片继承标签，避免误以为只改差异；用户明确选择这一批照片的新标签集合。
- 说明文案明确保存后会把这些照片替换成同一组标签。

### 保存行为

保存时前端对已选照片按顺序循环请求：

```text
PUT /api/session-album/photos/:photoId/tags
body: { tagKeys: selectedTagKeys }
```

保存语义：

- 对每张选中照片，替换为同一组 `tagKeys`。
- 空 `tagKeys` 允许保存，表示把这批照片清空标注，回到仅上传者可见。
- 保存期间锁定相册操作，展示 `正在保存标注...` 或包含进度的状态。
- 保存完成后关闭面板、退出多选模式、清空选择并刷新相册。

## 数据和接口

不新增接口。

沿用：

- `GET /api/sessions/:id/album`
- `GET /api/sessions/:id/album/people`
- `PUT /api/session-album/photos/:photoId/tags`

前端新增状态只存在于相册页组件内：

- `selectionMode`
- `selectedPhotoIds`
- `bulkTagging`

`albumBusy` 需要把批量保存状态纳入锁定条件，防止上传、删除、单张标注和批量标注同时发生。

## 错误和边界

- 用户未选择照片时不能开始批量标注。
- 当前筛选下没有可标注照片时，隐藏多选入口。
- 保存过程中如果某一张失败，继续保存剩余照片；最终提示 `部分照片标注失败`。
- 如果全部失败，提示 `标注保存失败`，仍刷新相册以同步服务端状态。
- 如果部分成功，刷新后以服务端返回为准，不做本地猜测合并。
- 保存过程中用户不能关闭面板后继续触发其他相册操作。
- 退出多选模式时清空选中照片，避免跨筛选或跨刷新误操作。

## 测试与验收

1. 小程序相册页存在多选入口和批量操作栏。
2. 多选模式下只能选择 `can_tag` 照片。
3. 多选模式下点击照片切换选择，不触发预览。
4. 未选照片时批量标注按钮不可用。
5. 批量标注复用人物分组，包括 NPC 角色。
6. 保存时前端循环调用现有单张标签接口。
7. 每次请求都发送同一组 `tagKeys`，并符合替换整组标签语义。
8. 保存期间相册操作被锁定。
9. 部分失败时给出部分失败提示，并刷新相册。
10. 单张标注、删除、上传、隐私设置和过滤不被破坏。

## 自检

- 本设计没有新增后端接口，符合“前端循环保存最安全”的决策。
- 批量保存语义明确为替换同一组标签，不是追加。
- 权限仍由现有单张接口逐张校验，前端不可选只是体验优化，不作为安全边界。
- 失败处理不假设本地状态可靠，最终以刷新后的服务端相册为准。
- 设计范围只覆盖小程序相册批量标注，没有扩大相册隐私或管理能力。

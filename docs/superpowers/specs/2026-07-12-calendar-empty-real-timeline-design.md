# 日历空状态复用真实时间轴设计

## 目标

纠正游客首页空状态：不再绘制独立的日期刻度轴，直接复用有车局时已经使用的真实时间轴结构和样式。

## 界面设计

- 空列表且加载结束时，渲染一个当天的真实 `day-band`。
- 时间轴必须复用现有 `timeline-rail`、`day-marker` 和 `day-card`，不得保留或新增平行的轴线、日期刻度、圆点样式。
- 当天节点沿用真实列表的“今”标记、墨绿色和尺寸。
- 当天卡片显示标题“今天还没有公开车局”、说明“下一场公开车局发布后，会出现在日期轴上”和“刷新车局”操作。
- 删除空状态内的“选择其他日期”；顶部已有日期选择按钮保持不变。
- 空列表继续隐藏“已显示全部车局”。
- 水墨底纹可以保留，但只作为当天空卡片的低对比背景，不参与时间轴绘制。

## 行为边界

- 刷新复用现有 `refreshCalendar()`，不请求登录。
- 非空列表、成员日历、同城列表、顶部归位和日期选择能力保持不变。
- 加载中和加载失败提示沿用现有逻辑。

## 验证

- 静态检查必须确认空状态包含真实时间轴类名，且不再包含 `calendar-empty-route-axis`、`emptyDateTicks` 或“选择其他日期”。
- D40 检查、小程序总检查和微信小程序构建通过。
- 微信开发者工具中确认当天空状态使用与真实车局列表相同的时间轴节点。

# 相册整册朋友圈使用 Skyline 页面截图设计

状态：待用户书面复核

日期：2026-07-24

## 1. 背景

相册整册分享给好友或群聊时已经省略 `imageUrl`，由微信使用当前页面的默认截图。
朋友圈分享仍由客户端 Canvas 生成方形拼图，并在失败时回退到静态图片。

微信的两个分享渠道默认行为不同：好友/群聊省略 `imageUrl` 时使用页面截图；
朋友圈省略 `imageUrl` 时使用小程序 Logo。因此，朋友圈要得到真正的页面截图，
必须先生成本地截图文件，再把该文件作为 `onShareTimeline` 的 `imageUrl`。

微信 Skyline 的 `snapshot` 组件可以把组件子树的渲染结果导出为图片。该组件从
基础库 3.0.1 开始支持，只支持 Skyline，并且子树不能包含原生组件。微信文档允许
把截图组件移到屏幕外渲染，但不能使用 `display: none` 或
`visibility: hidden`。

当前项目使用 uni-app `3.0.0-5000720260410001`。uni-app 在
`3.0.0-5010520260709001` 对应的稳定版本中增加了微信 Skyline `snapshot`
组件支持，因此本设计包含一次受控的编译器升级。

## 2. 决策

1. 仅相册整册分享统一使用页面截图。
2. 好友/群聊继续使用微信原生默认截图，不自行生成图片。
3. 朋友圈使用 Skyline `snapshot` 生成的本地页面截图。
4. 删除相册整册分享的 Canvas 拼图与静态图片兜底。
5. 截图失败、过期或运行环境不支持时，不开放朋友圈菜单，不回退到小程序 Logo、
   Canvas 或静态图。
6. 单张媒体、招募、首页、游后感和安全失败卡片保持现有封面策略。

## 3. 适用范围

本设计覆盖相册页的三种整册快照：

| 入口 | 好友/群聊 | 朋友圈 |
|---|---|---|
| 成员默认整册分享 | 微信当前页默认截图 | Skyline 页面截图 |
| 成员选择“全部”或“选中”后的主动快照 | 微信当前页默认截图 | Skyline 页面截图 |
| 公开相册接收者再次转发 | 微信当前页默认截图 | Skyline 页面截图 |

以下入口明确不变：

- 单张照片继续使用对应照片。
- 单个视频继续使用视频封面或现有安全封面。
- 招募分享继续使用招募卡片。
- 首页继续使用品牌图。
- 游后感继续使用首张照片或现有兜底图。
- 无效 token、未知按钮和不可分享状态继续使用安全失败卡片。

## 4. 编译器与渲染器兼容关卡

先把以下三项依赖统一升级到 `3.0.0-5010520260709001`：

- `@dcloudio/uni-app`
- `@dcloudio/uni-mp-weixin`
- `@dcloudio/vite-plugin-uni`

只为 `pages/session/album` 配置 Skyline 渲染器和 GlassEasel 组件框架，其他页面保持
WebView。项目当前微信基础库版本是 3.17.0，高于 `snapshot` 所需的 3.0.1。

升级后的第一道关卡只验证依赖和渲染兼容性，不删除旧 Canvas：

1. 微信小程序构建成功。
2. 编译结果中的相册页使用 Skyline 和 GlassEasel。
3. 成员相册和公开相册可以加载。
4. TDesign 操作按钮、筛选器、弹层、瀑布流、图片预览和视频入口没有阻断性问题。
5. `snapshot` 节点能够导出本地 PNG 临时文件。

如果任一关卡失败，停止后续删除工作并报告具体不兼容点；不得在没有朋友圈截图能力的
状态下移除现有生产路径。

## 5. 页面截图表面

相册滚动列表继续负责正常浏览。截图组件放在滚动区域之外的屏幕外位置，使用当前整册
分享 token 对应的公开安全数据渲染一个方形页面截图表面。

截图表面必须：

- 使用与公开相册首屏相同的信息层级、样式和真实页面组件：分享人、角色、剧本、店家、
  日期、媒体数量及首屏照片网格。
- 使用当前整册快照允许公开的媒体，最多渲染首屏实际可见数量；不得读取成员私有列表中
  不属于当前快照的照片。
- 只包含 `view`、`text`、`image` 等可被 Skyline 截取的节点，不包含 `video` 等
  原生组件；视频位置显示其安全海报图。
- 保持 1:1 方形尺寸，适配朋友圈封面比例。
- 位于屏幕外但仍参与渲染，不使用 `display: none`、`visibility: hidden` 或
  `v-if=false` 隐藏。
- 复用相册公开首屏的数据投影和样式，不重新实现 Canvas 拼图算法。

该表面是当前公开相册页面首屏的 DOM 截图，不是照片拼贴海报。正常页面内容发生语义
变化时，截图表面使用相同数据重新渲染。

## 6. 截图状态与数据流

新增独立的整册截图状态：

- 当前 token 和截图内容摘要。
- 截图生成代次。
- `preparing`、`ready`、`failed` 状态。
- 当前本地临时图片路径。
- 当前请求是否仍属于有效分享上下文。

流程如下：

1. 后端返回有效整册分享 token 和公开安全媒体数据。
2. 页面安装分享快照并渲染屏幕外的 Skyline 截图表面。
3. 在下一次 Vue 更新和 Skyline 布局完成后，通过选择器获取 `Snapshot` 实例。
4. 调用 `takeSnapshot({ type: "file", format: "png" })`。
5. 仅当 token、截图摘要和请求代次仍有效时，保存 `tempFilePath` 并标记朋友圈就绪。
6. `onShareTimeline` 返回原有标题和 query，并把该临时路径作为 `imageUrl`。

好友分享 readiness 只依赖有效 token。朋友圈 readiness 同时依赖有效 token 和当前
截图路径。

成员默认整册、成员主动快照和公开相册再转发复用同一截图控制器；不同 token 或不同
公开媒体摘要不得复用截图。

## 7. 失效、清理与并发

以下事件必须使旧截图立即失效：

- 分享 token 变化、撤销或过期。
- 默认整册被新的媒体语义替换。
- 成员创建新的选中/全部快照。
- 公开相册切换到另一个分享 token。
- 登录身份或分享权限变化。
- 页面卸载。

迟到的 `takeSnapshot` 结果只能被当前代次接收。被替换的临时文件使用微信文件系统
尽力删除；删除失败不得阻塞浏览或新的截图生成。

滚动、图片渐进加载等纯展示变化不自动触发无限截图。截图摘要发生变化或当前截图尚未
成功时才准备新截图，以避免频繁导出。

## 8. 菜单与失败行为

- 好友菜单：有效 token 就绪后立即开放。
- 朋友圈菜单：只有当前 Skyline 截图成功后才开放。
- 截图运行时不可用、导出失败、图片尚未具备安全预览或请求过期时，朋友圈菜单保持隐藏。
- `onShareTimeline` 被异常调用但没有当前截图时，返回不可分享状态，不得提供 Logo、
  Canvas 或静态 `imageUrl`。
- 朋友圈截图失败不得影响好友/群聊分享。
- 页面不自动滚动，也不为截图改变用户正在浏览的可见界面。

## 9. 删除范围

兼容关卡和 Skyline 截图测试通过后，删除仅服务于相册整册封面的客户端能力：

- 相册页中的两个隐藏 Canvas 节点。
- 相册页 Canvas runtime、导出、缓存、协调器和 cover recipe 消费流程。
- 整册 friend/timeline Canvas URL 与 prepared 状态中已无消费者的字段。
- 整册朋友圈的静态图片 fallback。
- 只验证旧 Canvas 拼图的测试和静态门禁。

后端暂时可以继续返回 `cover_recipe`，本次不修改相册分享 token、权限、审核、撤销或
过期协议。确认没有其他客户端消费者后，再单独删除后端冗余字段。

## 10. 测试与验证

自动化验证至少包括：

1. 依赖版本和相册页 Skyline/GlassEasel 配置检查。
2. 截图控制器的成功、失败、并发、过期结果和临时文件清理测试。
3. 三种整册朋友圈 payload 都必须使用当前 Skyline 临时路径。
4. 三种整册好友 payload 都必须省略 `imageUrl`。
5. 没有有效截图时朋友圈菜单保持隐藏。
6. 相册页不再包含 Canvas 节点、Canvas 准备调用和整册静态 fallback。
7. 单媒体、招募和安全失败 payload 仍保留原有明确封面。
8. 相册相关 Node 测试、D55/D56 门禁、小程序结构检查和生产构建全部通过。

真机/开发者工具验收覆盖：

- 成员默认整册分享。
- 成员选择全部后的整册分享。
- 成员选择部分媒体后的整册分享。
- 公开相册再次转发。
- iOS 与 Android 微信客户端。
- 图片和视频混合相册。
- 截图生成失败时好友仍可分享、朋友圈入口不可见。

## 11. 验收标准

1. 相册整册分享到好友或群聊时，卡片展示当前相册页面截图。
2. 相册整册分享到朋友圈时，方形封面展示公开相册页面首屏截图。
3. 朋友圈不再出现 Canvas 拼图、大面积白底、静态票根或小程序 Logo。
4. 截图只包含当前分享 token 允许公开的数据。
5. 单媒体、招募、首页、游后感和安全失败卡片行为不变。
6. 相册浏览、上传、筛选、标注、下载、图片预览和视频查看在 Skyline 下可用。

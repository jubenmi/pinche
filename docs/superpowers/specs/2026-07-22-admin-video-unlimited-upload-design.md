# 管理员视频无业务时长与大小上限设计

## 目标

系统管理员（`system_admin`）上传相册视频时，不受应用层的视频时长和文件大小上限限制。管理员仍必须先成功压缩视频，且 MP4 格式、内容审核、相册权限和上传安全校验保持不变。

## 当前行为

- 小程序只向 `system_admin` 开放视频上传入口。
- 小程序拒绝超过 60 秒的视频，并拒绝超过 100MB 的最终上传文件。
- 小于或等于 20MB 的视频可能跳过压缩；压缩失败时可能回退上传原文件。
- API 创建视频记录时拒绝超过 60 秒的时长。
- COS 上传意图、COS 请求头校验、对象检查和本地 multipart 上传都使用 100MB 上限。
- 普通用户没有视频上传入口，API 也会拒绝普通用户创建或上传视频。

## 权限策略

| 身份 | 视频上传权限 | 时长上限 | 文件大小上限 | 压缩要求 |
| --- | --- | --- | --- | --- |
| `system_admin` | 允许 | 无业务上限 | 无业务上限 | 必须成功压缩 |
| 普通用户 | 禁止 | 不适用 | 不适用 | 不适用 |
| 未来会员 | 本次不实现 | 计划为 60 秒 | 计划为 100MB | 后续需求决定 |

本次不增加会员身份、会员入口、会员 API 或会员策略分支。后续开放会员视频上传时，应在权限策略层新增 60 秒 / 100MB 规则，而不是重新给底层 MP4 元数据校验器增加全局上限。

## 小程序上传流程

1. 保持 `canUploadVideo` 只允许 `system_admin`。
2. 选择视频后读取原始时长、尺寸和文件大小；时长和大小必须是可读取的正数，但不设置业务最大值。
3. 每次都调用 `wx.compressVideo`，不再按 20MB 阈值决定是否跳过。
4. 只有满足以下条件才把压缩产物交给上传流程：
   - 压缩接口存在且调用成功；
   - 返回可读取的压缩文件路径，且不是原始文件路径；
   - 压缩产物大小为正数；
   - 产物未命中现有的异常压缩结果检测。
5. 压缩不可用、失败、返回原文件或结果异常时终止上传，提示管理员先在本地压缩或更换设备，不得回退上传原文件。
6. 删除 60 秒拒绝分支和 100MB 拒绝分支。确认弹窗继续展示实际时长和压缩后预计上传大小。
7. `wx.chooseMedia.maxDuration` 只描述微信现场拍摄能力，不作为相册文件上传限制。代码命名和注释应明确区分“拍摄时长”与“上传时长”，避免未来重新引入 60 秒业务校验。

## API 与存储边界

### 角色授权

管理员视频上传意图、COS 上传授权、本地上传路由和视频记录创建继续要求 `system_admin`。普通用户的行为保持为拒绝，不因底层元数据校验放宽而获得上传能力。

### 时长

视频记录创建继续要求 `durationSeconds` 为正整数，但不再应用 60 秒最大值。数据库现有 `INT UNSIGNED` 是物理数据类型边界，不定义产品业务上限。

### 文件大小

底层视频元数据校验继续要求文件大小为正的 JavaScript 安全整数，但不再应用 100MB 最大值。这样管理员的大文件可以通过，同时 MP4 内容类型和文件头校验仍独立生效。

COS 上传意图不再给管理员视频返回 `maxBytes`。客户端已有逻辑会把缺失或非正数的 `maxBytes` 解释为没有应用层上限。COS 授权仍校验请求方法、对象键、内容类型、内容长度和禁止覆盖请求头。

本地 multipart 解析器继续采用流式写入临时文件，不把完整视频加载到内存。管理员视频调用允许省略文件和请求总大小上限，但继续限制 multipart 头部大小、只接受一个视频 part、检查结束边界，并在失败时清理临时文件。

### 格式和审核

以下行为保持不变：

- 文件名必须为 `.mp4`；
- MIME 必须为 `video/mp4`；
- 文件内容必须包含 MP4 `ftyp` 文件头；
- 上传前检查内容安全 intake 状态；
- 创建媒体后继续提交视频内容审核；
- 未审核通过的内容不作为公开可播放媒体发布。

## 错误处理

- 无法读取时长或压缩后大小：终止并提示换一个视频或先本地处理。
- 微信压缩能力不可用或压缩失败：终止并提示必须先压缩，不上传原文件。
- 压缩结果异常：终止并保留现有异常压缩提示语义。
- COS 或本地上传失败：保留现有重试和失败反馈，不把无大小上限误报为允许无限资源。
- 设备存储、微信客户端、网络、COS、磁盘和数据库类型仍可能形成物理限制；这些失败按实际基础设施错误反馈，不表现为 60 秒或 100MB 业务限制。

## 测试设计

### 小程序

- 管理员选择超过 60 秒的视频可以进入压缩流程。
- 管理员选择超过 100MB 的视频可以进入压缩流程。
- 小于 20MB 的视频也调用压缩。
- 压缩失败、压缩能力不可用、返回原路径或异常产物时不会调用上传 API。
- 合法压缩产物即使超过 100MB 也可以继续上传。
- 普通用户的 `canUploadVideo` 仍为 false，并继续走图片选择流程。

### API

- `system_admin` 可以创建时长超过 60 秒的视频记录。
- 非管理员创建视频记录仍返回权限错误。
- 超过 100MB 的 MP4 元数据可以通过管理员上传链路的大小校验。
- 非 MP4 MIME、扩展名或文件头仍被拒绝。
- 管理员 COS intent 不再携带 `maxBytes`，授权仍校验权限、内容安全状态和安全请求头。
- 无大小上限的 multipart 流仍校验结构并在错误时清理临时文件。
- 内容审核入口与媒体创建事务顺序保持现有测试覆盖。

## 非目标

- 不开放普通用户视频上传。
- 不实现会员账号或会员视频上传。
- 不取消视频压缩。
- 不取消 MP4、权限或内容审核校验。
- 不保证突破微信客户端、设备、网络、COS 或操作系统的物理限制。

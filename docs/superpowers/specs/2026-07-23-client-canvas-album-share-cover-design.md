# 客户端三图分享封面设计

## 决策

公开相册分享封面从服务端 JPEG 合成改为微信小程序本地 Canvas 合成，并将封面数量收紧为最多三张：1 张单图、2 张双图、3 张及以上精选三图。

## 原因

服务端合成要读取多张图片对象、进行像素分析和 Sharp 渲染，即使有缓存，首次请求仍有显著 CPU、内存、存储读取与失败面。用户当前相册已在本地持有的预览图直接用于三图 Canvas；仅在本地预览缺失时才取 640px 缩略图，可进一步降低等待与网络开销，也让封面留出更大的视觉空间。

## 边界

服务端仍负责公开快照、角色/审核/隐私复核和短期缩略图授权，并只下发 `cover_recipe`。客户端优先按媒体 ID 复用当前页面的本地预览，缺失才用短期缩略图；不访问原图、不扩大候选集合。任何失败均使用已有本地降级图，正文分页与封面生成解耦。

详细验收见 [D55 设计](../../../specs/d55-client-canvas-album-share-cover/design.md)。

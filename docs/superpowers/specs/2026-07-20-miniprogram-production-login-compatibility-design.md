# 小程序生产包登录兼容性修复设计

## 目标

修复微信小程序生产构建中点击登录即报错的问题，增加能够覆盖生产构建产物和真实微信登录链路的冒烟测试，并在验证通过后完成受控 CI 发布及体验版覆盖上传。

## 已确认故障

- 开发构建在微信开发者工具中可以完成真实 `wx.login`、生产 API 换码、令牌保存和车局数据加载。
- 当前上传使用的 `dist/build/mp-weixin` 生产包点击登录时抛出 `Cannot read property 'showModal' of undefined`。
- 报错发生在登录确认弹窗阶段，登录 HTTP 请求尚未发送，因此生产 API、数据库、Redis 和微信换码服务不是本次故障点。
- 生产压缩产物把 `tdesignFeedback` 模块引用解析为可能为空的对象，而登录公共模块直接读取其 `showModal` 属性，没有保护模块对象本身。

## 修复方案

在 `apps/miniprogram/src/utils/api.js` 中把 TDesign feedback 改为命名空间依赖，并通过小型安全适配函数调用：

1. TDesign feedback 模块和目标方法可用时，继续使用现有 TDesign 弹窗、Toast 和 ActionSheet。
2. 模块对象或方法不可用时，回退到 `uni.showModal`、`uni.showToast` 和 `uni.showActionSheet`。
3. 平台 API 也不可用时，调用传入的失败回调，让现有登录流程能够结束，而不是抛出未捕获异常。
4. 只替换 `api.js` 中的 feedback 调用，不改登录协议、后端接口、页面结构或其他页面的展示逻辑。

不采用以下方案：

- 不关闭生产压缩；这会掩盖依赖访问缺陷并扩大上传包体。
- 不重构全部页面弹窗；本次问题集中在公共登录路径，全面重构会扩大回归范围。

## 测试设计

### 自动回归测试

增加面向安全 feedback 适配层的单元测试，至少覆盖：

- TDesign 方法存在时优先调用 TDesign。
- TDesign 模块为空时回退到对应 `uni` 方法。
- 两层实现都缺失时触发失败回调且不抛出异常。

生产构建后增加产物检查，确认登录模块包含安全访问和平台回退，防止开发源码测试通过但压缩产物重新出现直接属性访问。

### 生产包真实登录冒烟

提供本地发布门禁脚本：

1. 生成 `dist/build/mp-weixin` 生产包。
2. 通过微信开发者工具官方自动化能力打开该生产包。
3. 从首页点击登录并确认。
4. 等待真实 `wx.login` 和生产 API 登录完成。
5. 断言首页显示已登录身份，并确认开发者工具没有出现 `showModal` 未定义错误。

该冒烟测试依赖本机已登录的微信开发者工具，因此作为发布前本地门禁运行；CI 运行单元测试和生产产物检查。

## 发布流程

1. 只提交本次修复、测试、脚本和必要依赖，不纳入工作区既有无关改动。
2. 本地完整验证通过后推送 `develop`，等待 CI 成功。
3. 把同一修复晋级到 `main`，等待 CI 成功。
4. 把已验证的 `main` 晋级到 `publish`，等待 CI 成功。
5. 使用验证通过的生产包覆盖上传微信体验版，并记录版本号和上传结果。

任何分支 CI 失败时停止后续晋级；生产包真实登录失败时不得上传体验版。

## 验收标准

- 生产构建点击登录不再抛出 feedback 模块未定义错误。
- 微信开发者工具中的生产包能够完成真实登录并加载当前用户数据。
- 新增自动测试在 CI 中通过。
- `develop`、`main`、`publish` 的 CI 均成功。
- 新体验版覆盖上传成功，且代码来源与通过验证的发布提交一致。

## 范围外事项

- 不提交正式审核，不发布正式线上版本；本轮止于覆盖体验版。
- 不修改微信登录后端协议或账号数据。
- 不处理与本次登录故障无关的工作区文件。

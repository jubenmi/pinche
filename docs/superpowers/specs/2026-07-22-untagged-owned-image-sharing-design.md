# 本人未标注图片便捷分享设计

日期：2026-07-22

## 结论

本人上传、尚未标注的图片可以进入本人发起的公开相册分享和单图分享，但系统不把它们写成“其他”标签，也不在图片上保存长期的“允许未标注公开”状态。

未标注图片只在一次明确的分享快照内获得公开资格。整册分享先进入只读预览，让分享者看到接收者实际会看到的媒体范围；用户可以直接分享，也可以进入可选的内容调整模式排除不想公开的项目。单图分享已经展示了准确目标，因此继续在全屏预览中直接使用微信原生分享按钮。

未标注状态、普通成员相册可见性和“其他”标签语义保持不变。

## 背景

当前公开分享要求媒体至少存在一个角色或场景标签。即使图片是分享者本人上传，只要尚未标注，也不能进入整册或单图分享。用户必须逐张选择“其他”，才能分享普通风景、道具或暂时不需要人物分类的图片。

强制点击“其他”并不能真正证明画面里没有其他玩家：用户仍可能为了继续分享而直接选择“其他”。它增加了操作，却没有形成可靠的隐私判断。

直接把未标注图片写成“其他”同样不可取：

- “尚未处理”不等于“用户明确确认是风景/主线外照片”。
- “其他”标签会影响同车成员可见性，自动写入会扩大日常相册范围。
- 错误标签会污染角色筛选、统计和后续隐私判断。

因此，分享资格必须与标签分类分离；隐私确认应建立在“用户看见准确分享范围”上，而不是建立在一次形式化标注上。

## 目标

1. 本人上传的未标注图片无需逐张标为“其他”即可分享。
2. 整册分享前展示准确的公开快照，不让用户只对一个抽象数量盲目确认。
3. 用户可直接接受默认范围，也可按需排除具体媒体。
4. 未标注图片在普通成员相册中继续只有上传者可见。
5. “其他”继续表示用户主动选择的场景类标签，不被系统自动写入。
6. 上传者、被标注者、审核状态和媒体状态继续作为动态公开门禁。
7. 整册分享和单图分享使用同一套服务端资格函数。
8. 适配微信原生同步分享机制，不设计无法在一次点击内完成的异步确认流程。

## 非目标

- 不自动识别人脸、角色或图片内容。
- 不允许分享者带出其他用户上传的未标注图片。
- 不把未标注视频纳入本次放宽范围。
- 不改变明确标注为“其他”、NPC、角色或席位的既有语义。
- 不改变未标注筛选、批量标注、下载或内容审核流程。
- 不让未标注图片自动成为微信分享卡片封面。
- 不保存跨分享长期复用的图片级公开授权。

## 方案比较

### 方案一：自动写入“其他”标签

上传后或分享前创建 `other:session` 标签。操作最少，但会污染标签并扩大成员相册可见性，不采用。

### 方案二：图片级长期公开授权

为媒体保存 `allow_untagged_public_share`。它能减少重复确认，但会形成用户难以感知、难以撤销、可能长期残留的隐藏状态；标签保存、隐私开关和旧链接恢复之间也容易产生解释困难，不采用。

### 方案三：分享快照级资格 + 分享前预览

未标注图片只在当前公开快照中被明确记录，图片本身不变化。整册先预览准确范围，再通过原生按钮分享；需要时再调整内容。授权随分享快照在 30 天后失效，不跨快照永久复用。该方案最符合现有 D48 固定快照模型，作为最终方案。

## 核心语义

### 未标注与“其他”

- `tags.length === 0` 继续表示“未标注”。
- `other:session` 只在用户主动选择“其他”并保存时存在。
- 普通相册卡片继续把未标注图片显示为“待标注”。
- 公开接收页不展示“待标注”“隐式其他”或内部授权状态。

### 分享快照级资格

公开分享记录继续保存最终 `media_ids`，并新增本快照内的未标注图片事实，概念字段为 `implicit_untagged_media`。每个条目至少保存：

- `media_id`
- 创建快照时的 `tag_version`

这组数据只说明“该图片在本次分享快照中以分享者本人上传的未标注图片身份进入”，不代表图片已被外发，不回写媒体，也不影响其他分享。真正外发仍发生在用户预览后点击微信原生分享按钮时。

`snapshot_digest` 必须覆盖 `implicit_untagged_media`，避免含义不同的快照被错误复用。

### 标签版本

图片媒体具有单调递增的 `tag_version`，默认从 `0` 开始。每次保存标签都在同一事务中递增，包括标签结果与保存前相同、添加标签或清空标签。

如果当前 `tag_version` 与分享快照中记录的版本不同，该图片不得再以“本次确认的未标注图片”身份出现在旧分享中。这样可以避免图片被标注人物、再清空标签后自动恢复到旧公开链接。

图片后来存在有效标签时，也不得借用旧的未标注资格；新分享必须按当前标签重新计算。

## 分享资格

### 本人未标注图片

一张图片只有同时满足以下条件，才能以未标注身份进入当前用户发起的快照：

1. 媒体类型为图片，不是视频。
2. `uploader_user_id` 等于分享者用户 ID。
3. 图片属于当前车局。
4. 图片状态为 `active`。
5. 审核状态为 `approved` 或 `approved_legacy`。
6. 当前标签为空。
7. 上传者的 `allow_uploaded_visible = true`。
8. 创建快照时记录当前 `tag_version`。

未标注图片不依赖分享者席位标签，因为“本人上传”已经建立分享者关联；它不能为其他成员建立分享资格。

### 已标注媒体

存在标签的媒体完全沿用当前 D48/D50 规则：

- 分享者上传或标注了分享者席位，才满足关联条件。
- 上传者关闭公开时排除。
- 任一绑定真实用户的标签关闭被标注可见时排除。
- 席位与用户关系无法验证时关闭式排除。
- 视频必须处于 `ready`。

### 排序与数量

整册快照继续最多包含 30 项媒体，其中视频最多 3 项。默认优先级调整为：

1. 分享者上传且标注分享者席位。
2. 其他人上传且标注分享者席位。
3. 分享者上传的其他已标注合规媒体。
4. 分享者上传的未标注合规图片。

同一优先级继续按稳定创建时间和 ID 排序。未标注图片不会挤掉更明确的角色相关内容。

单图分享指定 `focusMediaId` 时，目标媒体先进入快照，再按现有规则补齐完整相册范围；目标不合规时关闭式失败。

## 用户流程

### 整册分享入口

成员相册新增明确的 `预览并分享` 操作。新客户端在成员相册中隐藏直接绕过预览的整册微信分享菜单；旧客户端仍可按旧规则分享已标注媒体，不会自动获得未标注范围。

点击 `预览并分享` 后：

1. 客户端请求服务端按默认规则生成公开快照，并显式声明允许纳入本人未标注图片。
2. 服务端在事务内重新计算资格、排序和数量，保存快照及 `implicit_untagged_media`。
3. 客户端进入现有公开相册只读模式的“分享预览”状态。
4. 预览展示与接收者相同的媒体内容和顺序。
5. 页面显示：`本次分享共 X 项，其中 N 张是你上传的未标注图片。未标注不等于无人出镜；如有不适合公开的其他玩家，请调整内容或先标注。`
6. 页面提供 `调整分享内容` 和微信原生 `分享给好友或群`。

预览本身取代额外确认弹窗。用户看见实际范围后点击原生分享按钮，即表示确认当前快照。

### 调整分享内容

`调整分享内容` 是可选操作，不是分享前置步骤：

- 默认预选当前快照中的全部媒体。
- 用户可以取消选择不想公开的项目，不需要给它们增加标签。
- 只能选择当前用户有权分享且通过审核的媒体。
- 最多选择 30 项，视频最多 3 项。
- 保存后服务端重新验证所选 ID，并创建或复用与新范围一致的快照。
- 保存后的展示顺序继续使用现有公开相册的稳定时间顺序，不使用用户点击勾选的先后顺序。
- 服务端不得因为某个选择项失效而自动替换成用户未选择的其他媒体。

调整模式可复用现有相册多选视觉和底部工具栏，但使用独立的 `share` 目的状态，不能复用或修改批量标注选择结果。

### 单图分享

全屏预览已经让用户看到准确目标，因此不再增加确认弹窗或中间页面：

- 本人未标注图片打开预览后，可以准备带 `focusMediaId` 的分享快照。
- token 准备完成后显示现有微信原生分享按钮。
- 分享按钮附近使用简短说明：`未标注，仅在你主动分享后公开`。
- 非本人上传的未标注图片保持不可分享。
- 目标失效时保持当前 D50 关闭式失败，不回退其他媒体。

### 接收页

- 接收者继续进入现有公开相册只读模式。
- 不展示内部标签、未标注资格、上传者 ID 或标签版本。
- 整册接收页展示快照剩余有效媒体；单图接收页先只展示目标媒体。
- 公开快照全部失效时展示统一不可查看状态，不提示具体隐私用户。

## 微信分享行为

现有 `onShareAppMessage` 只能同步返回已准备好的 payload，因此不在该回调内弹确认或发起异步授权。

- 整册：在分享预览页提前准备好 token 和封面，再展示原生分享按钮及右上角菜单。
- 单图：在全屏预览期间提前准备目标 token，ready 后才把图标切换为原生分享按钮。
- token 未准备好、准备失败或媒体失效时返回安全失败 payload。
- 朋友圈分享与好友/群分享使用同一快照、同一资格和同一封面集合。

## 微信分享封面

未标注图片即使已经进入公开快照，也不得成为微信卡片封面。封面继续按现有安全优先级选择：

1. 分享者本人上传、明确标注本人席位且不含其他真实玩家的图片。
2. 分享者本人上传、只含明确安全场景标签的图片，例如 NPC、场景或“其他”。
3. 没有安全图片时使用品牌兜底封面。

这项限制只降低外部入口曝光，并不把相册正文描述为“私密”；分享链接仍可能被转发或发到朋友圈。

## 动态复核

分享快照固定媒体范围，但每次公开列表、封面和媒体文件读取仍动态复核：

- 状态不再是 `active`：排除。
- 审核不再通过：排除。
- 上传者关闭 `allow_uploaded_visible`：排除。
- 已标注媒体触发任一真实用户隐私否决：排除。
- 未标注快照项的标签不再为空或 `tag_version` 变化：排除，旧快照中不自动恢复。
- 视频不再为 `ready`：排除。

上传者以后重新打开 `allow_uploaded_visible` 时，仍沿用现有 D48 动态恢复语义；标签版本发生变化的未标注快照项除外，它必须通过新快照重新确认。

## 服务端边界

- 扩展统一公开资格函数，使其接收分享快照上下文，而不是读取媒体级长期授权。
- 空标签分支只接受快照中记录了匹配 `media_id + tag_version` 的本人图片。
- 整册默认选择、自定义选择和单图 `focusMediaId` 共用同一资格函数。
- 自定义媒体 ID 必须由服务端重新校验车局、成员、上传者关系、审核、标签、隐私、类型和数量，不能信任客户端。
- 默认快照和自定义快照都在事务内完成选择、版本记录、摘要和分享记录创建。
- `snapshot_digest` 同时覆盖媒体顺序、隐式未标注条目、封面媒体、分享者和席位。
- 公开 DTO 不返回 `implicit_untagged_media`、`tag_version` 或原始标签。
- 封面选择器继续要求非空安全标签，不读取未标注快照资格。

## 错误与竞态

- 生成预览时部分默认候选失效：跳过失效项，使用剩余合规默认候选；全部失效时返回 `ALBUM_PUBLIC_SHARE_EMPTY`。
- 保存用户自定义选择时部分项目失效：返回通用“部分内容状态已变化，请重新确认”，不静默补入其他媒体。
- 非本人尝试把未标注图片加入快照：权限校验失败，不创建分享。
- 自定义选择超过 30 项或视频超过 3 项：客户端阻止，服务端再次拒绝。
- 标签保存与快照创建并发：事务读取到的标签及版本必须一致；不一致时重算或返回重试错误。
- 快照创建后标签发生变化：公开读取因 `tag_version` 不匹配排除该未标注项目。
- 单图目标准备期间失效：返回 `ALBUM_PUBLIC_SHARE_MEDIA_UNAVAILABLE`，不得回退其他媒体。
- token 或封面准备失败：留在成员页或预览页并允许重试，不开启原生分享入口。

## 测试与验收

### 服务端单元测试

1. 本人上传、未标注、已批准图片可进入显式允许未标注的快照。
2. 同一图片不能由其他分享者带入快照。
3. 未标注视频不能进入该分支。
4. 上传者关闭公开、媒体未批准或非 active 时不能进入。
5. 已标注媒体继续执行完整标签隐私规则。
6. 未标注项只有 `media_id + tag_version` 与快照匹配时才能公开读取。
7. 标签保存递增版本后，旧快照中的隐式未标注项立即排除且不自动恢复。
8. 未标注项不进入封面候选。
9. 默认排序中未标注图片低于所有已标注关联内容。
10. 整册、用户自定义范围和单图分享使用一致资格结果。

### API 与事务测试

1. 新客户端可请求包含本人未标注图片的默认分享预览。
2. 旧客户端未声明新能力时继续只获得旧版已标注范围。
3. 自定义选择混入他人未标注图片、其他车局媒体或未批准媒体时被拒绝。
4. 自定义选择按现有公开相册的稳定时间顺序展示，不使用勾选顺序，也不静默增加未选媒体。
5. 快照摘要包含隐式未标注媒体及标签版本，语义不同的快照不复用。
6. 标签并发变化不会让错误版本进入快照。
7. 隐私、删除、审核或标签变化后，旧分享动态失效相应项目。
8. 单图目标不合规时关闭式失败。

### 小程序测试

1. 普通相册仍把相关图片显示为“待标注”，其他成员仍不可见。
2. `预览并分享` 展示与最终接收页一致的默认媒体和顺序。
3. 预览明确显示总数和未标注图片数量。
4. 用户无需调整即可直接使用原生分享按钮。
5. 调整内容可以取消选择，不会修改标签、下载选择或普通相册可见性。
6. 超过数量限制时界面给出明确提示，服务端同样拒绝。
7. 单图预览本人未标注图片时可直接准备原生分享，非本人未标注图片不可分享。
8. 未标注图片不会成为微信封面；没有安全封面时使用品牌兜底图。
9. token 未就绪或失败时不显示可执行的原生分享按钮。
10. 新客户端成员页不能通过右上角菜单绕过整册分享预览。

## 发布与兼容

- 数据库只新增快照字段和标签版本，不新增媒体级长期公开开关。
- 旧分享记录的 `implicit_untagged_media` 为空，继续执行原有“未标注不可公开”规则。
- 旧客户端调用现有接口但不声明 `includeOwnedUntaggedImages` 时继续得到旧版范围。
- 服务端先上线兼容字段、统一资格和读取复核，再上线小程序分享预览与调整流程。
- 回滚新小程序后，旧客户端不会创建新的未标注快照；已有快照按服务端动态门禁继续有效至过期。
- 分享快照继续沿用现有 30 天有效期和撤销机制。

## 决策摘要

“未标注”与“其他”保持两个不同概念。便利性通过默认纳入本人未标注图片解决；隐私通过准确预览、可选排除、仅本人上传、动态门禁、标签版本和安全封面限制保证。授权属于一次分享快照，不属于图片本身，也不会永久影响后续分享或普通成员相册。

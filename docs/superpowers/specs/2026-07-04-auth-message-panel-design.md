# 顶部消息面板设计

日期：2026-07-04

## 目标

在微信小程序顶部个人信息栏增加轻量消息提示，让车头能直接发现“待审核上车申请”，并从展开面板进入车头管理页处理。

## 范围

第一版只展示车头待审核申请，不做站内私信、不展示聊天未读、不新增消息表。

## 交互

- 已登录车头如果存在待审核申请，顶部个人信息栏右侧显示“消息 N”提示。
- 点击“消息 N”展开面板。
- 面板展示每个有待审核申请的车局、待审数量、店家和时间。
- 点击消息项跳转到 `/pages/session/manage?id=<session_id>`。
- 面板内提供刷新和收起。
- 点击个人信息栏其他区域继续保留现有个人信息弹层能力。

## 数据流

- `AuthIdentityBar` 复用现有接口 `GET /api/users/me/sessions?limit=50`。
- 只取 `pending_signup_count > 0` 的车局生成消息项。
- 申请审核完成后，页面重新显示或手动刷新会同步最新数量。
- 请求失败不影响登录条或个人信息编辑，只在面板内显示轻量错误。

## 实现边界

- 新增一个纯函数 helper 负责把车局列表转换为消息项，便于测试。
- `AuthIdentityBar` 负责拉取、展示、刷新和跳转。
- 不修改后端接口和数据库。

## 验证

- 新增脚本检查消息 helper 的过滤、计数和文案回退。
- 检查顶部个人信息栏包含消息徽标、展开面板、刷新和管理页跳转。
- 运行小程序静态检查和新增检查脚本。

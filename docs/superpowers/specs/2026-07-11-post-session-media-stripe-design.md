# 游戏结束相册媒体竖条设计

日期：2026-07-11

状态：已确认，实施计划已生成

实施计划：[2026-07-11-post-session-media-stripe.md](../plans/2026-07-11-post-session-media-stripe.md)

## 摘要

首页“我的车局”日历继续使用左侧黄、酒红、绿三色竖条，但调整游戏结束后的颜色语义：

- 尚未到发车时间：黄色。
- 已到发车时间且相册没有有效照片或视频：酒红色，视为“炸车”。
- 已到发车时间且相册至少有一张有效照片或一个有效视频：绿色。

“游戏结束”继续沿用现有 `start_at <= 当前时间` 判定，不新增完成按钮或持久化状态。

## 背景

`SessionCalendar.vue` 当前只要车局已到发车时间，就会在 `calendarStripeTone` 中无条件返回绿色。这会让没有任何相册沉淀的已结束车局也显示为绿色，无法表达“玩完但无人上传媒体”的炸车状态。

车头列表 `listMySessions` 已统计活跃相册记录，但字段仍沿用 `photo_count` 命名，而且会把失败视频一起计入。参与者列表 `listMySignups` 尚未返回相册媒体计数，因此不能只靠前端改一个判断完成一致展示。

## 目标

1. 让车头和已上车玩家看到一致的三色竖条。
2. 已结束且没有有效相册媒体的车局显示酒红色。
3. 任意成员上传有效照片或视频后，在下一次首页数据刷新时自动显示绿色。
4. 保持现有取消、拒绝等失败状态的酒红色语义。

## 非目标

- 不新增“游戏结束”“炸车”数据库字段或人工操作。
- 不改变状态标签文字、相册提示文字、卡片背景或页面布局。
- 不改变相册权限、上传、转码、删除和隐私规则。
- 不改变同城和游客列表；这些列表只展示尚未发车的招募车局，继续使用黄色。

## 颜色判定

竖条按以下优先级计算：

| 优先级 | 条件 | 颜色 |
| --- | --- | --- |
| 1 | 车局已取消，或玩家报名被拒绝/取消 | 酒红色 |
| 2 | 尚未到发车时间 | 黄色 |
| 3 | 已到发车时间，且有效相册媒体数为 0 | 酒红色 |
| 4 | 已到发车时间，且有效相册媒体数大于 0 | 绿色 |

有效相册媒体定义为 `session_album_photos.status = 'active'` 且：

- 图片：`media_type = 'image'`；图片登记后即为 `ready`。
- 视频：`media_type = 'video'` 且 `processing_status <> 'failed'`；`processing` 和 `ready` 都表示上传已经成功，因此计入。

删除媒体后按当前数据实时重算。如果最后一个有效媒体被删除，已结束车局重新显示酒红色。失败视频不使竖条变绿；之后重新上传成功的照片或视频会使其变绿。

## 后端数据

`listMySessions` 和 `listMySignups` 都新增数值字段 `album_media_count`，使用同一有效媒体条件计算。

- `listMySessions` 保留现有 `active_album_photo_count` 和 `photo_count`，避免影响取消车局前的相册清理等既有逻辑；新颜色只读取 `album_media_count`。
- `listMySignups` 通过按车局关联的计数子查询返回 `album_media_count`，使参与者卡片使用相同口径。
- API 返回值把计数归一化为非负数字；缺失或非法值在前端按 0 处理。

不新增迁移或持久化派生状态。媒体上传、处理失败或删除后，列表查询自然得到最新计数，避免维护两份状态。

## 小程序逻辑

把三色判定提取为可独立测试的纯函数，输入只包含：

- 是否为取消/拒绝等失败状态；
- 是否已到发车时间；
- `album_media_count`。

`SessionCalendar.vue` 继续负责从车局或报名记录组装这些输入，并把纯函数结果绑定到现有 `session-stripe`。现有 `amber`、`red`、`green` 样式和酒红色值保持不变。

上传完成返回首页后，现有首页 `onShow`/刷新流程重新请求列表；不引入推送或本地猜测颜色。若用户停留在首页，可通过已有下拉刷新取得最新状态。

## 错误与兼容处理

- 旧后端未返回 `album_media_count` 时，前端按 0 处理，因此已结束车局保守显示酒红色。
- 媒体计数查询失败时沿用现有列表请求错误处理，不单独伪造绿色状态。
- 取消/拒绝优先级高于媒体计数，即使相册已有媒体仍显示酒红色。
- 本次不复用 `has_review` 或评价数量；只有照片或视频能让竖条变绿。

## 测试

按测试驱动流程先增加失败用例，再实现最小逻辑。至少覆盖：

1. 未到发车时间且无媒体时为黄色。
2. 已到发车时间且无媒体时为酒红色。
3. 已到发车时间且有照片时为绿色。
4. 已到发车时间且有处理中视频时为绿色。
5. 已到发车时间但只有失败视频时，后端计数为 0，竖条为酒红色。
6. 取消、报名拒绝或取消时，无论媒体计数为何都为酒红色。
7. 车头列表和参与者列表都返回同口径的 `album_media_count`。
8. 项目现有小程序检查、API 语法检查和小程序构建继续通过。

# 自动内容审核能力与降级拦截设计

## 目标

为全站图片、视频和文本建立统一的内容发布判定：腾讯云自动审核能力可用时，一律自动审核；能力不可用时，由系统管理员配置的兜底拦截开关决定直接发布还是隐藏。默认保持当前直接发布行为。

本期不提供人工审核、人工批准或人工拒绝流程。

## 范围

- 所有新提交的图片入口，例如头像、车局相册和评价配图。
- 所有新提交的视频入口。
- 所有受内容安全约束的新建或修改文本入口。
- 系统管理员后台的“内容安全”设置页。

不扫描或改变历史内容；历史内容继续遵循原有可见性规则。

## 判定模型

系统为 `image`、`video`、`text` 三个类型分别评估自动审核能力。能力可用的最低条件是该类型所需的腾讯云凭证、地域、策略和客户端健康检查均正常。

每种内容的发布判定固定为：

| 自动审核能力 | 无自动审核时拦截开关 | 处理结果 |
| --- | --- | --- |
| 可用 | 任意 | 提交自动审核；仅通过后发布。审核中的内容保持隐藏。 |
| 不可用 | 关闭（默认） | 直接发布，保持未接入审核时的既有行为。 |
| 不可用 | 开启 | 阻拦新内容，不创建公开内容或公开 URL。 |

已经成功提交自动审核的内容，若结果超时、回调异常或服务中断，必须保持隐藏；不能再退回“能力不可用时直接发布”的分支。该开关只处理“提交前已判定无审核能力”的降级情形。

## 配置与权限

系统管理员后台新增“内容安全”页面，仅 `system_admin` 可读取或修改。

- 一个总开关：`没有自动审核能力时阻拦内容`，默认关闭。
- 三个类型开关：图片、视频、文本。总开关开启时，分别控制对应类型在无自动审核能力时是否拦截。
- 页面展示每种类型的当前能力状态：可用、未配置或不可用；不展示密钥或其他敏感配置。
- 每次变更记录操作者、前后值和时间。

类型开关不用于关闭已可用的腾讯云自动审核；一旦对应能力可用，内容必定进入自动审核。

## 统一组件与服务边界

API 新增统一的内容安全判定服务，向各业务入口提供两步接口：

1. `getModerationCapability(contentType)`：仅判断自动审核能力是否可用。
2. `resolvePublicationPolicy(contentType)`：根据能力状态与管理员设置返回 `auto_moderate`、`publish_directly` 或 `block_unavailable`。

图片、视频和文本提交服务统一调用该服务，不在小程序或管理后台前端自行决定发布状态。各类型保留其技术处理职责：图片和视频上传/转码，文本保存/更新；发布可见性只消费统一策略结果。

## 用户体验与错误处理

- 自动审核中：提示“内容正在安全审核”，不提供公开访问地址。
- 自动审核不通过：提示“内容未通过安全审核”。
- 自动审核能力不可用且拦截开关开启：提示“内容安全服务暂未就绪，暂时无法发布，请稍后再试”。
- 自动审核能力不可用且开关关闭：无额外提示，按当前成功发布体验处理。

客户端不展示腾讯云标签、评分、策略标识、密钥或内部错误详情。

## 测试与验收

覆盖图片、视频和文本三种类型的以下情形：

1. 自动审核能力可用时，不论开关取值均进入审核，未通过前不可公开。
2. 自动审核能力不可用且类型开关关闭时，保持直接发布。
3. 自动审核能力不可用且类型开关开启时，阻拦发布且无公开 URL。
4. 审核请求已创建后发生超时或错误时，内容保持隐藏，不走直接发布降级。
5. 非系统管理员无法读取或写入内容安全设置。
6. 每个实际图片、视频和文本入口都复用统一服务，不能绕过判定。

同时保留已有媒体访问门禁测试，确保待审、失败和拒绝内容不会通过列表、分享、预览、下载或直连 URL 泄露。

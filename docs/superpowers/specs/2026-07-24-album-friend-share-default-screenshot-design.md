# 相册好友分享使用微信默认截图设计

状态：待用户书面复核

日期：2026-07-24

## 1. 背景

当前相册好友/群聊分享由小程序端 Canvas 生成 `1000×800` JPEG，并在
`onShareAppMessage` 返回值中显式提供 `imageUrl`。生产截图显示，Canvas 使用原图
尺寸计算裁剪区域，却实际加载最大约 `640×640` 的缩略图，导致源裁剪范围越界；
有效图像只占预期槽位的一小部分，其余透明区域导出 JPEG 后成为大面积白底。

相册公开页首屏已经包含分享人、角色、剧本、店家、日期、照片数量和照片网格，
适合作为好友分享卡片的页面截图。微信在 `onShareAppMessage` 未提供 `imageUrl`
时使用当前页面的默认截图，因此好友封面不再需要额外生成。

## 2. 目标

1. 整册相册、用户选中的相册快照和公开相册再次转发到好友/群聊时，省略
   `imageUrl`，由微信使用当前页面截图。
2. 分享标题、公开相册 token、落地路径、快照隐私、审核、撤销和过期能力保持不变。
3. 好友分享不再等待 Canvas 封面准备完成；有效 token 和可分享页面状态就绪后即可开放。
4. 朋友圈继续使用现有 `1000×1000` 专用封面。
5. 招募分享、单张照片/视频分享和失败关闭卡片继续使用现有明确封面，不改为页面截图。

## 3. 非目标

- 不删除朋友圈 Canvas 生成能力。
- 不改变相册页面视觉布局或强制滚动位置。
- 不把手机系统状态栏、微信导航栏等整机截图保存为固定素材。
- 不修改后端相册快照、封面 recipe 或媒体 URL 接口。
- 不改变分享标题和分享落地页。

## 4. 分享入口行为

| 分享入口 | 好友/群聊封面行为 |
|---|---|
| 成员通过右上角菜单分享全部相册 | 返回 `title` 和 `path`，省略 `imageUrl` |
| 成员分享当前选中的相册快照 | 返回 `title` 和 `path`，省略 `imageUrl` |
| 公开相册接收者再次转发 | 返回 `title` 和 `path`，省略 `imageUrl` |
| 单张照片或视频分享 | 保留当前媒体封面 |
| 招募分享 | 保留安全票根封面 |
| token 缺失或入口不可用 | 保留失败关闭卡片及其安全封面 |
| 朋友圈分享 | 保留现有独立方形封面 |

好友分享 payload 必须真正省略 `imageUrl` 属性；不得返回空字符串、`null` 或静态
兜底图来模拟省略。

## 5. 客户端状态与数据流

成员进入相册后，现有预热流程继续创建默认整册分享 token。默认好友分享 readiness
只依赖有效 token，不再依赖 `defaultAlbumShareFriendCoverPrepared`。用户创建选中
快照后，活动分享 readiness 同样只依赖有效 token。

公开相册加载成功并持有 `albumShareToken` 后，好友分享菜单可以开放。朋友圈菜单仍
需要 `shareTimelineCoverPrepared`，因此两个渠道的 readiness 独立。

为降低本次变更风险，后端仍可返回 `cover_recipe`，客户端朋友圈仍可消费该 recipe。
好友渠道不再等待或消费 Canvas 结果；后续确认无其他消费者后，才能单独评估删除冗余
好友封面状态和绘制代码。

## 6. 错误与降级

- token、路径或分享上下文无效时，不得退回默认页面截图，继续使用现有失败关闭 payload。
- 朋友圈 Canvas 失败时继续使用朋友圈静态兜底图，不影响已就绪的好友分享。
- 页面照片尚未加载完成时，微信可能截取到现有占位状态；本期不新增截图等待协议。
- 分享期间不得为了截图自动滚动页面，以免用户界面跳动。

## 7. 测试

1. 单元测试锁定好友默认截图 payload 只含 `title`、`path`，没有 `imageUrl` 自有属性。
2. 页面静态测试覆盖默认整册、活动快照和公开相册三个好友分享分支均使用默认截图。
3. 测试确认招募、单媒体和失败关闭 payload 仍保留 `imageUrl`。
4. 测试确认朋友圈 payload 仍保留独立 `imageUrl`，朋友圈 readiness 仍由方形封面控制。
5. 运行相册分享相关 Node 测试、项目小程序静态检查和小程序构建。

## 8. 验收

在微信开发者工具或体验版打开相册首屏，分别从成员相册和公开相册触发“发送给朋友”：

- 分享标题与落地路径正确；
- 卡片使用当前页面默认截图，不再出现三张小图漂浮在大白底上的 Canvas 结果；
- 接收者打开后仍只能看到分享快照允许公开的内容；
- 单媒体、招募和朋友圈分享外观及落地行为没有变化。

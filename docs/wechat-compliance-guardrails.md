# 微信小程序合规护栏

更新日期：2026-06-11

本文用于约束产品、文案、交互和开发实现，目标是在设计阶段避开微信小程序平台明确禁止或高风险的规则点。后续所有MVP功能都应先过本文清单。

## 2. 产品定位合规口径

小程序对外定位：

```text
剧本杀发车/拼车工具
情感本组局工具
座位价格与线下锁位状态记录工具
```

小程序对外不定位为：

```text
恋爱陪伴平台
陪玩/陪聊/交友平台
现金红包或返现平台
抽奖、网赚、拉新奖励平台
剧本内容分发平台
```

公开页面只展示组局决策信息，不展示剧透、现实陪伴承诺、身体接触承诺、现金奖励承诺。

## 3. 硬性禁区

### 禁止诱导分享

不能出现：

- 分享后才能报名、解锁价格、查看座位、生成海报。
- 转发后获得现金、红包、积分、优惠券、优先锁座、信用加分、抽奖机会。
- “帮我转发一下”“分享到群有补贴”“转发得低价”等明示或暗示文案。
- 红点、抖动、弹窗、倒计时等诱导用户点击分享。
- 通过客服消息、订阅消息或弹窗反复催促用户转发。

允许出现：

- 用户主动点击“分享给好友”。
- 车头复制招募文案。
- 已上车玩家自愿分享“还差几人”的车况。
- 后台记录分享来源和转化，但不把分享次数兑换权益。

### 禁止红包、网赚和返现化

不能出现：

- 平台红包、现金红包、转发领红包。
- 分享得现金、邀请得补贴、拉人返现。
- 抽奖、概率性收益、提现门槛。
- “补贴后倒返钱”“负实付到账”“上车还赚钱”。

MVP规则：

```text
payable_price >= 0
不做平台红包
不做现金奖励
不做返现提现
不做补贴倒付
```

“出补/吃补”在产品里只表达为车内座位价格调整，用于计算实付价，不包装成平台奖励。

### 禁止色情、陪伴服务或暧昧承诺

公开页面不能使用或暗示：

- 现实恋爱陪伴。
- 包恋爱、包牵手、包小黑屋、包亲密互动。
- 私下陪伴、线下约会、陪聊陪玩。
- 色情、挑逗、身体交易相关表达。

推荐公开表达：

```text
情感沉浸位
演绎互动位
指定DM/NPC
剧本内互动
现场边界以玩家自愿为准
```

“恋陪”“爱D”可以作为行业研究和内部标签保留，但不建议作为小程序名称、简介、分享标题、海报主标题或默认公开标签。

### 禁止侵犯隐私、肖像和知识产权

不能出现：

- 未经授权展示DM/NPC真实姓名、头像、照片、视频。
- 未经授权使用剧本封面、剧本正文、角色文本、发行素材。
- 公开展示玩家手机号、微信号、真实姓名、报名备注、边界备注。
- 要求用户输入微信账号密码。
- 将用户联系方式卖给店家、DM/NPC或第三方。

MVP策略：

- 剧本只保存本名和无剧透标签。
- 店家和剧本由系统管理员录入为未认领资料；车头缺资料时只提交新增资料申请。DM/NPC手填名称只作为本车快照，未绑定本人微信用户前不展示头像、主页或“官方认证”。
- 联系方式只对车头在审核场景可见。
- 分享图和海报使用无隐私模板。

### 禁止类目不符和隐藏功能

不能出现：

- 用工具、日历、资讯等类目包装实际组局交易服务。
- 提审版本隐藏真实功能，上线后再打开。
- 不给审核人员完整测试路径。
- 用H5、插件或外链绕开小程序规则。
- 强制用户下载App、跳公众号、加个人号才能继续主流程。

MVP策略：

- 提审前根据实际服务选择类目。
- 首页两次点击内可到达核心服务。
- 审核测试账号可体验建车、报名、审核、分享主链路。

### 禁止滥用订阅消息

不能出现：

- 订阅后给奖励。
- 不订阅就不能报名或建车。
- 下发营销消息、拉新消息、无关广告。
- 下发与用户订阅时理解不一致的内容。

允许场景：

- 申请上车后，订阅审核结果。
- 锁座后，订阅开车前提醒。
- 车头发布后，订阅新申请提醒。

用户拒绝订阅时，主流程必须继续。

### 内容安全必须前置

所有用户可输入并可能展示的内容，都要进入内容安全流程：

```text
昵称
本名
店名
DM/NPC名称
备注
边界备注
招募文案
海报文案
```

MVP要求：

- 文本发布前做敏感词和内容安全检测。
- 图片上传如开放，必须做图片内容安全检测。
- 高风险词命中后提示用户改写，不能直接公开展示。
- 后台保留人工处理入口。

## 4. 推荐文案替换

| 内部黑话 | 公开页面建议 | 原因 |
| --- | --- | --- |
| 恋陪 | 情感沉浸位 / 演绎互动位 | 避免被理解为现实陪伴服务 |
| 爱D | 指定DM/NPC | 降低暧昧和饭圈化风险 |
| 出补/吃补 | 座位价格调整 / 实付价 | 避免红包、返现、网赚联想 |
| 免单 | 实付0元 | 避免抽奖、福利诱导感 |
| 返现 | 禁用 | 容易变成现金奖励或网赚 |
| 红包 | 禁用 | 红包类账号和诱导分享风险 |
| 小黑屋/牵手 | 剧本内互动边界 | 避免身体接触承诺 |
| 押金 | 线下锁位押金状态 | 明确MVP不代收不担保 |

## 5. MVP功能调整

必须调整：

- 分享贡献只做统计，不给权益。
- 订阅消息只做业务提醒，不做营销和奖励。
- 座位实付最低为0，不允许负数和倒返钱。
- 押金只记录线下状态，不由平台代收。
- “补贴”只在车内配平，不做平台补贴和现金红包。
- 分享海报不包含联系方式、真实姓名、转账信息。
- 公开文案默认使用“情感沉浸位”“指定DM/NPC”“实付价”。

暂不做：

- 平台红包。
- 邀请奖励。
- 分享排行榜。
- DM/NPC明星榜。
- 随机抽奖。
- 平台代收押金。
- 站内公开评论区。

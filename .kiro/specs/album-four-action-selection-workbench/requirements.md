# 相册四操作与批量选择工作台需求

## 目标

在不重做相册页面、瀑布流和既有权限体系的前提下，把相册顶部操作区收敛为 `分享`、`下载`、`标注`、`招募` 四个双字图标按钮，并让分享和下载在点击后直接进入批量选择模式。用户可以在同一选择模式中执行“处理选中内容”或“处理全部符合条件的内容”，不再经过“全部 / 多选”动作选择层，也不再经过分享预览页。

## 术语

- **四操作区**：相册标题和隐私按钮下方的一行四个等宽操作按钮。
- **分享选择模式**：`selectionModePurpose = "share"` 的相册多选状态。
- **下载选择模式**：`selectionModePurpose = "download"` 的相册多选状态。
- **标注选择模式**：`selectionModePurpose = "tag"` 的现有批量标注状态。
- **分享全部**：分享该车局相册中提交时所有符合现有公开资格的媒体，不受当前筛选和当前勾选影响。
- **分享选中**：只分享用户在分享选择模式中明确勾选、且服务端复核后仍符合公开资格的媒体。
- **下载全部**：下载该成员完整相册中所有当前允许下载的图片，不受当前筛选和当前勾选影响。
- **下载选中**：只下载用户在下载选择模式中明确勾选的可下载图片。
- **招募**：进入现有邀请玩家认领角色流程；它不是相册公开分享，也不授予相册读取权限。
- **活动分享快照**：当前相册页面生命周期内，最近一次由“分享全部”或“分享选中”成功生成的公开相册快照。

## 界面需求

1. 相册标题、上传入口、隐私入口、相册筛选、角色筛选和照片瀑布流必须保持现有位置与语义。
2. 现有 `预览并分享`、`全部下载`、`多选下载`、`批量标注` 顶部按钮必须替换为一行四个等宽按钮。
3. 四个按钮必须按从左到右的顺序显示：`分享`、`下载`、`标注`、`招募`。
4. 四个按钮文案必须都是两个汉字，不得在普通状态下显示“预览分享”“全部下载”“多选下载”“批量标注”或“邀请认领”。
5. `分享` 必须使用三节点连接或等价的分享图标，不得复用标题上传入口的向上箭头。
6. `下载` 必须使用向下下载图标。
7. `标注` 必须使用标签图标，并继续作为墨绿色实心强调按钮。
8. `招募` 必须使用添加成员或人物加号图标。
9. `分享`、`下载`、`招募` 使用浅色描边按钮；四个按钮的高度、宽度、图标尺寸、图文间距和圆角必须一致。
10. 普通相册状态下四操作区与筛选区之间只使用现有轻量分隔和间距，不新增卡片、说明区或预览区。
11. 进入任一选择模式后必须隐藏普通四操作区，避免普通操作与底部批量操作同时出现。

## 分享需求

1. 点击 `分享` 必须直接进入分享选择模式，不得先弹出“全部分享 / 多选分享”动作层。
2. 分享选择模式必须显示可选状态、已选数量和取消入口。
3. 分享选择模式底部只能有两个业务操作按钮：`分享全部` 和 `分享选中`。
4. `分享全部` 必须使用次级样式；`分享选中` 必须使用主按钮样式。
5. 未选择任何媒体时，`分享选中` 必须禁用；`分享全部` 仍可用。
6. `分享选中` 必须显示当前选择数量，例如 `分享选中（6）`。
7. `分享全部` 必须显示当前客户端已知的可公开数量，例如 `分享全部（18）`；服务端返回的最终数量是权威结果。
8. 分享选择数量不得设置产品级固定上限；用户可以选择当前相册中任意数量的可公开媒体。
9. 分享选择必须同时支持符合公开资格的图片和已就绪视频。
10. 切换“全部 / 上传 / 我的 / 待标”筛选或角色筛选时，分享选择模式和已经选择的 ID 必须保留。
11. `分享全部` 必须忽略当前筛选和当前勾选，由服务端在提交时重新计算全部符合现有公开资格的媒体。
12. `分享选中` 必须提交用户选择的完整 ID 集合；任一选择项失效时整个请求失败，不得静默删除、替换或扩大分享范围。
13. 分享服务端必须继续执行成员资格、确认席位、审核状态、视频就绪状态、上传者隐私、被标记用户隐私和作者私有内容规则。
14. 本功能不得修改“符合公开资格”的业务定义；只改变用户选择范围和入口交互。
15. 分享快照必须固定媒体 ID 集合；公开读取时仍须重新检查删除、审核撤回和隐私收紧。
16. 分享快照中的媒体数量不得再受现有 30 项产品上限限制；封面候选仍最多使用 9 项。
17. 分享内容的公开展示顺序必须使用相册现有稳定时间顺序，不使用用户勾选顺序。
18. 点击 `分享全部` 或有效的 `分享选中` 后，页面必须显示准备中状态并阻止重复提交。
19. 分享准备成功后不得进入媒体预览页；页面只显示轻量的“分享已准备好”状态和微信原生好友 / 群聊分享按钮。
20. 朋友圈分享继续遵循微信限制，通过右上角菜单触发，并使用同一个活动分享快照。
21. 活动分享快照准备完成前，右上角好友分享和朋友圈分享菜单必须隐藏或关闭式失败，不得回退到角色招募链接或旧快照。
22. 分享失败时必须保留当前选择，允许用户重试或取消。
23. 接收者必须进入公开只读相册；公开页不得显示上传、下载、标注、隐私、招募或认领角色入口。

## 下载需求

1. 点击 `下载` 必须直接进入下载选择模式，不得先弹出“全部下载 / 多选下载”动作层。
2. 下载选择模式必须显示可选状态、已选数量和取消入口。
3. 下载选择模式底部只能有两个业务操作按钮：`下载全部` 和 `下载选中`。
4. `下载全部` 必须使用次级样式；`下载选中` 必须使用主按钮样式。
5. 未选择任何照片时，`下载选中` 必须禁用；`下载全部` 仍可用。
6. `下载选中` 和 `下载全部` 必须显示对应数量。
7. 下载选择数量不得设置产品级固定上限。
8. 下载模式只允许选择现有规则允许下载的已发布图片；视频和无下载权限媒体保持不可选。
9. 切换相册筛选或角色筛选时，下载选择模式和已经选择的 ID 必须保留。
10. `下载全部` 必须下载完整成员相册中的所有可下载图片，不受当前筛选和当前勾选影响。
11. `下载选中` 必须从完整可下载集合解析选择 ID，不得只从当前筛选结果解析。
12. 下载前必须继续显示包含数量的确认提示，并沿用现有系统相册权限申请、逐张保存、进度和部分失败反馈。
13. 下载完成或用户取消后必须退出下载选择模式并清理下载选择 ID。

## 标注需求

1. 点击 `标注` 必须直接进入现有批量标注模式。
2. 标注只允许选择当前用户有权标注的照片。
3. 标注继续复用现有人物 / 角色选择面板和保存流程。
4. 普通状态按钮文案改为 `标注`，但批量模式内可继续使用完整文案 `批量标注` 或 `标注选中`。
5. 本功能不得改变标签替换语义、标注权限、NPC 角色规则或后端保存接口。

## 招募需求

1. 点击 `招募` 必须直接进入现有“邀请好友认领角色”页面，不进入媒体选择模式。
2. 新入口必须使用当前角色招募路径，不再使用仅为历史相册入口兼容保留的 `entry=album`。
3. 招募继续使用独立的 `join-invite-token`，不得复用 `albumShareToken` 或公开相册快照。
4. 招募只允许发送给微信好友或群聊，不允许分享到朋友圈。
5. 招募卡片必须使用现有安全车局 / 票根封面，不使用相册照片。
6. 接收者继续按车局的 `direct` 或 `review_required` 规则认领玩家座位或 NPC 角色。
7. 相册分享凭证不得用于角色认领；招募凭证不得读取公开或完整相册。

## 状态与并发需求

1. 分享、下载和标注三种选择模式必须相互排斥，同时只能存在一种 `selectionModePurpose`。
2. 从普通状态进入任一选择模式时必须清理上一用途的残留选择和面板状态。
3. 取消选择模式必须清理当前用途选择并恢复四操作区。
4. 页面隐藏、退出登录、切换账号或进入公开只读模式时必须清理未提交的选择和准备中状态。
5. `albumBusy` 期间不得进入新的选择模式、切换用途或重复提交。
6. 单媒体分享继续使用既有独立状态，不得覆盖相册页活动分享快照或批量选择结果。
7. 没有可公开媒体时，点击分享应提示暂无可分享内容。
8. 没有可下载照片时，点击下载应提示暂无可下载照片。
9. 没有可标注照片时，点击标注应提示暂无可标注照片。
10. 没有可认领角色或车局不再接受加入时，招募页必须展示实时不可用状态。

## API 与兼容需求

1. `POST /api/sessions/:id/album/share-token` 必须支持显式的全部分享请求和选中分享请求。
2. 全部分享请求使用明确的 `scope: "all"`；选中分享请求使用非空、去重、正整数 `mediaIds`。
3. `scope: "all"`、`mediaIds` 和现有 `focusMediaId` 三种输入必须互斥。
4. `mediaIds` 不设置产品级固定数量上限，但必须受全局请求体大小限制、当前车局实际媒体数量和逐项权限校验约束。
5. 未传新字段的旧客户端请求继续沿用现有兼容默认范围，不得因为新客户端的“分享全部”语义而无条件扩大旧分享。
6. 现有 `focusMediaId` 单媒体分享必须保持兼容。
7. 快照摘要必须覆盖完整媒体集合、封面集合、车局、分享者和席位；相同集合可以复用有效快照，不同集合不得误复用。
8. 数据库继续复用 `session_album_public_shares.media_ids` JSON 字段，不新增快照媒体明细表。

## 非目标

- 不重做相册标题、上传、隐私、筛选、瀑布流、单图预览或公开相册视觉。
- 不新增相册分享预览页。
- 不新增“全部 / 多选”动作选择弹层。
- 不改变公开资格、隐私、作者私有内容或审核规则。
- 不让视频进入下载流程。
- 不改变单媒体分享行为。
- 不改变角色认领资料门槛、上车审核或 NPC 自选规则。
- 不新增分享奖励、招募奖励、红包、排行榜或社交关系。
- 不重构无关相册、分享、登录或内容审核模块。

## 验收标准

1. 普通相册操作区只显示 `分享`、`下载`、`标注`、`招募` 四个双字图标按钮。
2. 分享图标与上传图标明显不同，标注按钮保持墨绿色主视觉。
3. 点击分享直接进入分享多选，底部只显示“分享全部”和“分享选中”两个业务按钮。
4. 点击下载直接进入下载多选，底部只显示“下载全部”和“下载选中”两个业务按钮。
5. 分享和下载在未选择内容时禁用“选中”按钮，但“全部”按钮可用。
6. 分享全部覆盖提交时全部符合公开资格的媒体，且不受 30 项限制。
7. 分享选中支持任意数量并精确固定选择集合。
8. 下载全部覆盖完整相册可下载图片；下载选中支持跨筛选保留选择。
9. 分享准备完成后不显示预览页，好友 / 群聊和朋友圈使用同一个活动快照。
10. 标注继续进入现有批量标注流程；招募继续进入现有角色认领邀请流程。
11. 相册分享、单媒体分享和招募三种凭证互不越权。
12. 专项静态检查、API 测试、相册分享回归、小程序检查和微信小程序构建通过。

# D46 需求：自动内容审核能力与降级拦截

## 目标

为全站新图片、视频和文本提供统一的自动审核判断。腾讯云审核能力可用时必须自动审核；能力不可用时，系统管理员可以选择继续直接发布或阻拦新内容。

## 范围

- 所有新图片入口，包括头像、车局相册和评价配图。
- 所有新视频入口。
- 所有受内容安全约束的文本创建与编辑入口。
- 系统管理员后台的“内容安全”设置。

不含人工审核、申诉、历史内容扫描和自动封号。

能力状态以进程启动时的 provider 配置为准，生产预演提供发布前确认，不实现逐请求动态 provider health。任务创建后的权限、额度或网络失败属于审核失败/重试路径，不属于本次提交的降级条件。

## 验收要求

1. 默认时腾讯云审核能力未配置、总开关和三个类型开关均关闭；图片、视频和文本必须与最早的直接发布逻辑完全一致。
2. image、video、text 的自动审核能力分别判断；任一类型能力可用时，不论开关状态都必须先自动审核后发布。
3. 某类型能力不可用时，只有总开关和该类型开关同时开启才阻拦；否则保持直接发布。
4. 审核请求一旦创建，待审、超时、错误和拒绝内容不得退回直接发布，也不得产生公开 URL。
5. 只有 system_admin 可读取或修改设置；变更记录操作者、前后值与时间。
6. 客户端不得绕过后端发布决策，也不得展示腾讯云策略、标签、分数或内部错误。
7. 定向测试覆盖三种类型的能力可用、默认直发、无能力拦截、审核后异常隐藏、权限和读取门禁；npm run check 通过。
8. 若真实 provider 事故需要让后续新提交进入 unavailable fallback，须先关闭对应 provider enabled 配置并重启/滚动发布（或使用平台维护）；DB 双开关不能把仍标记为已启用的能力动态判为不可用。
